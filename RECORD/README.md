# RECORD — Production Assets

Production files for The Record. Design in **Adobe InDesign** only — see [`../BUSINESS/design-environment.md`](../BUSINESS/design-environment.md) for why this is non-negotiable.

**Start here:** [`design-brief.md`](design-brief.md) — the complete, self-contained handoff for the designer/bindery (object spec, typography, file construction, exact page copy, proofing, open decisions).

## Folders
- [`Design/`](Design/) — **Claude Design package** (standalone HTML: cover spec, editions, interior specs, interactive page-through model)
- [`InDesign/`](InDesign/) — the single `.indd` source file (see structure below)
- [`PDF/`](PDF/) — export-ready and proof PDFs
- [`Fonts/`](Fonts/) — licensed font files used in the book
- [`Images/`](Images/) — any supplied imagery (kept minimal by design)
- [`Covers/`](Covers/) — cover artwork and mockups

## Edition
90-day edition. Front matter → Initiation Days 1–7 → 90 Daily Pages → a Weekly Escalation Page after every 7th daily.

## InDesign file structure
One file, not multiple. Sections in order:
1. Identity Standard
2. Non-Negotiable Standards
3. Initiation Sequence (Days 1–7)
4. Daily Page (master, ×90)
5. Weekly Escalation Page (master)
6. Close-Out (final page, after Day 90)
7. Carryover / Index (if added later)

Rules:
- Master pages for Daily + Weekly
- No freeform page-by-page layout
- Paragraph styles only — no local overrides
- No page numbers except the Day count (and Week count on weekly pages)
- Generous inner margin so writing near the gutter is comfortable

## File-prep checklist (before sending to bindery)
- Daily page locked
- Initiation pages included
- Identity Standard + Non-Negotiable Standards included
- 90 daily pages + weekly pages in correct order
- Close-Out page included as the final page
- No page numbers except Day/Week counts
- No decorative elements added by a designer

Binderies will often "helpfully" adjust things. Do not allow that.
